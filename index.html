ChatApp

ChatApp 是課堂上的期末專題內容
應用了課堂上學到的 portaudio 函式庫
使用 WinAPI 的 Mutex 以及 Thread 實現非同步聊天
同時為了支援中文，也用到了 Console 相關的 API 去顯示中文字

雖然功能陽春，但由於是 C++ ，需要處理更多東西
最初僅使用 getch 以及 SetConsoleCursorPosition 去移動游標並重畫界面
但因為 Big5 編碼有 2 Bytes 使得顯示並不順利
於是做了調整，判斷 DBCSLeadByte ，完成了中文字的輸入

程式本身分成 2 個 Channel 傳送資料：

  • Message Channel
  • Audio Channel

音訊上比較難處理
現有的方式是藉由 portaudio 讀取 200ms 的內容，再一次送出
Buffer 會緩存最多 10 秒的內容，超過就會開始蓋掉聲音

最初整體架構全都寫在 ChatApp 裡面
後來將 Console、Audio、Socket 分別抽出來，並提供接口
使得 ChatApp 本身可以更專注在自己的內容上

Wen!n

WebIn 是課堂上的專題內容
因為想要有一個學習系統的通知器，所以才會開始這項專案

可以管理腳本運行的時間以及腳本內容
腳本本身需要自行撰寫
預設提供 BeautifulSoup 以及簡單包裝過的 Http 類別進行爬蟲

腳本容被分割成幾個步驟：

  1. setup - 腳本初始化設定
  2. login - 登入
  3. listing - 列出要爬取的對象
  4. fetch - 爬蟲取得資料
  5. alive - 保持登入
setup、login、alive 可以選擇性忽略
只有 listing 跟 fetch 為必要流程

界面的部份採用 PySide2 ，對應 Qt5
簡單的做了基礎的管理界面以及通知區塊

資料的部份則使用 SQLite 儲存已經爬取過的通知，以及內容

Baksys

Baksys 是課堂上的期末專題
利用 file io 撰寫檔案結構，並將檔案做壓縮
最後會變成一個本地備份檔
而本地的備份檔又可以上傳到伺服器進行異地備份

專案本身分為核心部份的 baksys 與 app
baksys 本身負責備份檔案的生成以及解包
app 則負責 client-server 部份的程式以及指令列

程式還有提供腳本進行備份以及遠端同步
可以自行加入 cronjob 排程
